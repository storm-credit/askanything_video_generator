# AskAnything 업데이트 점검 마스터 플랜 (Phase 4 운영용)

이 문서는 최근 업데이트(검색 강화 + ElevenLabs + Kling + Remotion) 이후,
"컷 수 부족(3~4컷)"과 "최종 영상 다운로드 404" 문제를 빠르게 진단/복구하기 위한 운영 체크리스트입니다.

## 1) 가장 먼저 확인할 핵심 증상 분류

### A. 컷이 3~4개만 생성됨
- 원인 후보
  - 오래된 서버 프로세스가 떠 있어서 최신 `cutter.py` 프롬프트가 반영되지 않음
  - API 응답이 JSON 파싱 전후에서 잘리거나 비정상 종료
- 즉시 확인
  1. 서버 재시작 후 첫 요청 로그에서 "총 N컷"을 확인
  2. N이 6~10 범위인지 확인

### B. 다운로드 시 404
- 원인 후보
  - Remotion 의존성 미설치로 렌더 실패 (`npm install` 미실행)
  - 렌더 실패로 파일 자체 미생성
  - 백엔드 재시작 전 오래된 상태를 프론트가 참조
- 즉시 확인
  1. SSE 로그에 `DONE|assets/.../video/*.mp4`가 찍혔는지 확인
  2. 해당 경로에 파일이 실제 존재하는지 확인

## 2) 표준 점검 순서 (권장)

1. **프로세스 정리 후 백엔드 재시작**
   - 기존 `api_server.py`/uvicorn 프로세스를 완전히 종료
   - `api_server.py` 재기동
2. **Remotion 엔진 준비**
   - `remotion/`에서 `npm install`
3. **헬스체크**
   - `/api/generate` 1회 요청으로 컷 수/진행률/완료 이벤트 확인
4. **결과 경로 검증**
   - DONE 메시지에 나온 MP4 경로가 실제 생성되었는지 확인
5. **재현 테스트 2회**
   - 같은 주제 1회 + 다른 주제 1회 실행하여 일관성 확인

## 3) 운영 체크리스트 (실행 커맨드)

아래 명령을 순서대로 실행하면 대부분의 캐시/의존성 문제를 정리할 수 있습니다.

```bash
# 1) 백엔드 재시작
pkill -f "uvicorn|api_server.py" || true
python api_server.py
```

새 터미널에서:

```bash
# 2) Remotion 의존성 설치
cd remotion
npm install
```

새 터미널에서:

```bash
# 3) 렌더 산출물 생성 여부 확인
find assets -type f | rg "final_shorts\.mp4|final_video\.mp4|remotion_props\.json"
```

## 4) 보강 포인트 (안정화 우선순위)

1. **사전 점검 자동화**
   - 서버 시작 시 `remotion/node_modules` 존재 여부를 검사해 미설치면 명확한 오류를 즉시 반환
2. **파일 존재 검증 강화**
   - `DONE|...`를 내보내기 전에 최종 mp4 존재/크기 검증
3. **컷 수 품질 게이트**
   - 컷 수가 6 미만이면 재시도 또는 사용자 경고를 구조화해서 반환
4. **운영 로그 표준화**
   - 컷 수, 렌더 시작/종료, 출력 파일 경로를 JSON 로그로 남겨 장애 분석 시간을 단축

## 5) "동시버전 4"(Remotion v4)로 갈 때 좋은 점

질문하신 "더 좋은 코드가 있느냐"의 핵심은 **코드 품질 자체보다 렌더링 엔진의 표현력/확장성**입니다.

- **표현력 향상**: React 기반이라 텍스트 팝업, 하이라이트, 애니메이션 타이밍 제어가 쉽습니다.
- **유지보수성 향상**: 화면 로직을 컴포넌트 단위로 분리 가능해서 기능 추가가 빠릅니다.
- **쇼츠 친화성**: 호르모지 스타일 같은 동적 자막/강조 효과를 구조적으로 구현하기 좋습니다.
- **한계/주의점**: Node 의존성이 필요하고, 초기 설치가 안 되어 있으면 렌더가 실패합니다.

즉, v4는 "무조건 더 빠른 코드"라기보다,
**쇼츠 품질과 확장성(연출 자유도)을 올리는 선택**입니다.

## 6) 추천 실행 계획 (오늘 바로 적용)

- 1단계: 서버 재시작 + `remotion/npm install` 수행
- 2단계: 테스트 주제 2개로 생성 후 컷 수(6~10)와 다운로드 정상 동작 확인
- 3단계: 실패 시 DONE 경로 파일 존재 여부부터 역추적
- 4단계: 위 보강 포인트 1~2번을 코드에 먼저 반영(예방 효과 큼)
