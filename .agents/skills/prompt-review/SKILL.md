# Prompt Review — AskAnything

Review all LLM prompts in the pipeline for quality and cost efficiency.

## Prompts to Review

### 1. Cut Planning System Prompt (`modules/gpt/cutter.py`)
- **Location**: Korean prompt ~line 125-191, English ~line 193-258
- **Est. tokens**: ~1500-2000 (system) + ~200 (user) = ~2000/call
- **Temperature**: 0.75 (set for balanced creativity/consistency)

**Check:**
- [ ] No redundant instructions (each line adds unique value)
- [ ] Golden examples: exactly 3 cuts (Hook/Twist/Cliffhanger)
- [ ] JSON output format clearly specified
- [ ] Emotion tags listed: [SHOCK], [WONDER], [TENSION], [REVEAL], [CALM]
- [ ] Cut count instruction clear: "7~10컷"
- [ ] Image prompt rules: "NO text in image", vertical composition
- [ ] Channel identity injection only when channel is specified

### 2. Safety Fallback Prompt (`modules/utils/safety.py`)
- **Location**: `get_safety_fallback_prompt()` function
- **Purpose**: Replace blocked prompts with safe alternatives

**Check:**
- [ ] Fallback stages progressive (stage 0 → 1 → 2)
- [ ] MASTER_STYLE applied once (not doubled)
- [ ] Fallback still produces relevant images (not generic)

### 3. Image Prompts (generated by LLM)
- **Location**: Cut's `image_prompt` field
- **Prefix**: MASTER_STYLE from `modules/utils/constants.py`

**Check:**
- [ ] MASTER_STYLE is concise (no filler phrases)
- [ ] No "text", "words", "letters" in image prompt rules
- [ ] Vertical composition emphasized (9:16)

### 4. Video Motion Prompts (`modules/utils/constants.py`)
- **Location**: `get_motion_style()` function
- **Purpose**: Add camera motion instructions based on emotion

**Check:**
- [ ] Each emotion tag maps to appropriate camera motion
- [ ] Motion descriptions are video-engine-compatible (Veo3/Kling/Sora)
- [ ] Default motion style exists for unmatched emotions

### 5. Fact-check Context (`modules/gpt/search.py`)
- **Location**: Injected into user content as `[Fact-Check Reference]`
- **Est. tokens**: ~300-500 (3 search results)

**Check:**
- [ ] Context format is concise (title + url + content snippet)
- [ ] Not duplicating information already in the topic
- [ ] Cache working (1-hour TTL, same topic returns cached)

## Token Budget Summary
| Component | Tokens | Cost (GPT-4o) |
|-----------|--------|---------------|
| System prompt | ~1800 | ~$0.009 |
| User content + fact-check | ~500 | ~$0.0025 |
| Output (7-10 cuts JSON) | ~1500 | ~$0.015 |
| **Total per video** | **~3800** | **~$0.027** |

## When to Run
- After editing system prompts in cutter.py
- After changing MASTER_STYLE or motion styles
- When cut quality degrades (wrong count, poor prompts)
- When optimizing API costs
