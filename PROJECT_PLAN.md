### 현재 당면 과제 (운영 트러블슈팅 우선순위)
1. **컷 수 강제 정책 검증**
   - 목표: 기획 결과가 항상 6~10컷 범위인지 자동 확인
   - 점검: `modules/gpt/cutter.py` 프롬프트 + 출력 JSON 검증 로그
2. **Remotion 런타임 의존성 검증**
   - 목표: `remotion/` 의 node_modules 설치 상태를 빌드 전에 빠르게 감지
   - 점검: `npm --prefix remotion install`, `npx --prefix remotion remotion --help`
3. **산출물 경로/서빙 경로 정합성 검증**
   - 목표: 렌더 완료 파일과 API 다운로드 라우트가 동일 경로를 참조하도록 보증
   - 점검: `api_server.py`의 파일 반환 경로 + 실제 출력 파일 존재 체크
4. **서버 캐시/프로세스 잔존 문제 차단**
   - 목표: 구버전 프로세스가 포트를 점유해 404를 유발하는 케이스 제거
   - 점검: 서버 재기동 절차 문서화, 재시작 직후 스모크 테스트 수행

### 권장 점검 순서 (운영 체크리스트)
1) 백엔드 서버 완전 종료 → 재시작
2) `remotion` 의존성 설치/복구
3) 짧은 주제로 1회 생성 스모크 테스트
4) 컷 개수(6~10), 렌더 로그, 최종 파일 존재 여부 검증
5) 프론트 다운로드 링크로 실제 파일 수신 확인

### "v4"를 선택하면 뭐가 좋아지나?
- **기능 추가 자체가 아니라, 현재 파이프라인 안정성과 품질을 끌어올리는 운영 단계**입니다.
- 핵심 가치는 아래 4가지입니다.
  - 사실성(검색 근거)
  - 몰입감(TTS + 모션 컷)
  - 시청 지속률(동적 자막)
  - 실패 복구력(체크리스트 기반 점검)
- 즉, "더 좋은 코드가 있나"의 답은 **새 엔진을 또 갈아끼우는 것보다, 지금 붙인 엔진들을 일관되게 성공시키는 운영 코드/점검 체계가 v4의 본질**입니다.
